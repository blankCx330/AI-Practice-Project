## 部署前准备

### 1. 确保项目可以正常构建

```bash
# 安装依赖
npm install

# 构建生产版本
npm run build

# 预览构建结果
npm run preview
```

构建成功后，会在项目根目录生成 `dist` 文件夹，包含：
- `index.html` - 入口文件
- `assets/` - 打包后的 JS 和 CSS 文件

### 2. 检查构建产物

```bash
# 查看 dist 目录结构
ls dist

# 应该看到类似输出：
# index.html
# assets/
#   ├── index-xxxxx.js
#   └── index-xxxxx.css
```

## 方式一：GitHub Pages 部署

### 步骤 1：创建 GitHub 仓库

1. 登录 [GitHub](https://github.com)
2. 点击右上角 `+` → `New repository`
3. 填写仓库名称（如 `pomodoro-timer`）
4. 选择 `Public`（公开）
5. 点击 `Create repository`

### 步骤 2：推送代码到 GitHub

```bash
# 初始化 Git（如果还没有）
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit"

# 添加远程仓库
git remote add origin https://github.com/你的用户名/pomodoro-timer.git

# 推送到 GitHub
git push -u origin main
```

### 步骤 3：配置 vite.config.ts

修改 `vite.config.ts`，添加 `base` 配置：

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  base: '/pomodoro-timer/', // 改成你的仓库名
})
```

### 步骤 4：创建部署工作流

创建文件 `.github/workflows/deploy.yml`：

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build

      - name: Setup Pages
        uses: actions/configure-pages@v4

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: './dist'

      - name: Deploy to GitHub Pages
        uses: actions/deploy-pages@v4
```

### 步骤 5：启用 GitHub Pages

1. 进入仓库页面
2. 点击 `Settings` → `Pages`
3. 在 `Source` 下选择 `GitHub Actions`
4. 推送新代码触发自动部署

### 步骤 6：访问你的网站

部署完成后，访问：
```
https://你的用户名.github.io/pomodoro-timer/
```

## 方式二：Vercel 部署

Vercel 是最简单的部署方式，无需任何配置。

### 步骤 1：注册 Vercel

1. 访问 [vercel.com](https://vercel.com)
2. 使用 GitHub 账号登录

### 步骤 2：导入项目

1. 点击 `Add New` → `Project`
2. 选择你的 GitHub 仓库
3. 点击 `Import`

### 步骤 3：配置项目

- **Framework Preset**: Vite
- **Root Directory**: `./`
- **Build Command**: `npm run build`
- **Output Directory**: `dist`

点击 `Deploy`

### 步骤 4：等待部署完成

Vercel 会自动：
- 安装依赖
- 构建项目
- 部署到全球 CDN

### 步骤 5：访问你的网站

部署完成后，Vercel 会提供一个域名：
```
https://你的项目名.vercel.app
```

### 步骤 6：自定义域名（可选）

1. 进入项目 `Settings` → `Domains`
2. 添加你的自定义域名
3. 按提示配置 DNS 记录

## 方式三：Netlify 部署

### 步骤 1：注册 Netlify

1. 访问 [netlify.com](https://www.netlify.com)
2. 使用 GitHub 账号登录

### 步骤 2：创建新站点

1. 点击 `Add new site` → `Import an existing project`
2. 选择 GitHub，授权后选择你的仓库

### 步骤 3：配置构建设置

- **Build command**: `npm run build`
- **Publish directory**: `dist`

点击 `Deploy site`

### 步骤 4：等待部署完成

Netlify 会自动构建和部署。

### 步骤 5：访问你的网站

部署完成后，Netlify 会提供一个域名：
```
https://随机名称.netlify.app
```

### 步骤 6：配置 netlify.toml（可选）

创建 `netlify.toml` 文件：

```toml
[build]
  command = "npm run build"
  publish = "dist"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

## 方式四：自有服务器部署

### 步骤 1：构建项目

```bash
npm run build
```

### 步骤 2：上传 dist 文件夹

将 `dist` 文件夹上传到服务器的 Web 目录。

**使用 SCP 上传：**
```bash
scp -r dist/* user@your-server:/var/www/html/
```

**使用 FTP/SFTP 工具上传：**
- FileZilla
- WinSCP
- Cyberduck

### 步骤 3：配置 Nginx

创建 Nginx 配置文件：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/html;
    index index.html;

    # 处理 SPA 路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
}
```

重载 Nginx：
```bash
sudo nginx -s reload
```

### 步骤 4：配置 Apache

创建 `.htaccess` 文件：

```apache
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>
```

## 常见问题解决

### 问题 1：页面空白

**原因**：`base` 路径配置错误

**解决**：修改 `vite.config.ts` 中的 `base` 配置：
- GitHub Pages: `base: '/仓库名/'`
- Vercel/Netlify: `base: '/'` 或删除此配置
- 自有域名: `base: '/'`

### 问题 2：刷新页面 404

**原因**：SPA 应用需要服务器配置重定向

**解决**：
- **Nginx**: 使用 `try_files $uri $uri/ /index.html;`
- **Apache**: 使用 `.htaccess` 重定向规则
- **Vercel/Netlify**: 自动处理，无需配置

### 问题 3：localStorage 数据丢失

**原因**：浏览器清除了数据或使用了隐私模式

**解决**：
- 使用普通浏览器模式
- 检查浏览器是否阻止了 localStorage
- 考虑添加数据导出功能备份

### 问题 4：构建文件过大

**原因**：未进行代码分割

**解决**：在 `vite.config.ts` 中添加：
```typescript
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['react', 'react-dom'],
          charts: ['recharts'],
        }
      }
    }
  }
})
```

### 问题 5：跨域问题

**原因**：API 请求跨域

**解决**：
- 使用代理服务器
- 配置 CORS 响应头
- 使用 Vite 开发服务器代理
