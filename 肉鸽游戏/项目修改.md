# 肉鸽游戏 - 项目修改文档

## 项目概述

这是一个基于 React + TypeScript + Canvas 的肉鸽游戏（Roguelike），玩家控制角色在地图上移动，自动攻击敌人，收集经验和金币升级。

---

## 发现的问题列表

### 1. 近战武器发射弹幕问题 ✅ 已修复
**现状**: 所有武器（包括近战武器如剑、斧、匕首）都会发射投射物弹幕。
**文件位置**: `src/game/GameCanvas.tsx` 第31-100行 `handleAutoAttack` 函数
**问题描述**: 代码中对所有武器统一调用 `addProjectile()`，没有区分近战和远程武器。

### 2. 商店界面无法按ESC返回 ✅ 已修复
**现状**: 在商店界面按ESC键无响应。
**文件位置**: `src/game/GameCanvas.tsx` 第136-142行 `handleKeyDown` 函数
**问题描述**: ESC键处理只检查 `paused` 和 `playing` 状态，未处理 `shop` 状态。

### 3. 近战武器无挥动动画 ✅ 已修复
**现状**: 近战武器只是作为图标环绕玩家旋转，攻击时没有挥动动画。
**文件位置**: `src/game/GameCanvas.tsx` 第286-300行 `drawWeapons` 函数
**问题描述**: 武器渲染只是简单绘制图标，没有攻击动画状态机。

### 4. 远程武器无攻击动画 ✅ 已修复
**现状**: 远程武器（弓、法杖）攻击时没有拉弓/施法动画。
**文件位置**: 同上
**问题描述**: 所有武器渲染逻辑相同，没有武器类型区分的动画。

### 5. 武器未瞄准锁定的敌人 ✅ 已修复
**现状**: 武器只是环绕玩家旋转，不会指向目标敌人。
**文件位置**: `src/game/GameCanvas.tsx` 第286-300行 `drawWeapons` 函数
**问题描述**: 武器朝向基于固定角度计算，未使用攻击时获取的目标角度。

### 6. 切换目标时无转向动画 ✅ 已修复
**现状**: 武器切换目标时瞬间指向新目标，没有平滑转向。
**文件位置**: `src/game/GameCanvas.tsx`
**问题描述**: 缺少武器角度插值/缓动逻辑。

### 7. 敌人被击杀无死亡动画 ✅ 已修复
**现状**: 敌人血量归零后直接消失。
**文件位置**: `src/game/store.ts` 第142-146行
**问题描述**: 敌人死亡只是从数组中移除，没有播放死亡动画的过渡状态。

### 8. 不同武器弹幕和攻击方式相同 ✅ 已修复
**现状**: 所有武器发射相同的黄色圆形弹幕，速度、大小、效果一致。
**文件位置**: `src/game/GameCanvas.tsx` 第73-87行
**问题描述**: 投射物创建没有根据武器类型定制参数。

### 9. 近战武器无飞行动画 ❌ 待实现（新需求）
**现状**: 近战武器（剑、斧、匕首）使用范围伤害逻辑，没有飞行动画效果。
**文件位置**: `src/game/GameCanvas.tsx` 第80-111行 `isMelee` 分支
**问题描述**: 近战武器需要改为"飞向敌人"的攻击方式，类似回旋镖/飞刀效果。

**期望效果**: 
- 近战武器不再使用范围伤害
- 武器从玩家身边飞向目标敌人
- 飞行过程中对碰撞的敌人造成伤害
- 造成伤害后飞回玩家身边
- 支持穿透多个敌人（根据pierce属性）

## 技术细节说明

### 武器类型分类
```typescript
// 近战武器 - 飞向敌人
const MELEE_WEAPONS = ['sword', 'axe', 'dagger'];

// 远程武器 - 发射弹幕
const RANGED_WEAPONS = ['bow', 'magic'];
```

### 弹幕差异化参数
```typescript
const PROJECTILE_CONFIGS = {
  bow: { color: '#D2691E', speed: 12, size: 5 },
  magic: { color: '#9370DB', speed: 7, size: 8 },
};
```

### 敌人死亡动画状态
```typescript
interface Enemy {
  // ... existing fields
  isDying?: boolean;
  deathProgress?: number; // 0-1 动画进度
}
```

### 武器动画状态
```typescript
interface Weapon {
  // ... existing fields
  currentAngle?: number;  // 当前朝向
  targetAngle?: number;   // 目标朝向
  isAttacking?: boolean;
  attackProgress?: number;
}
```

## 注意事项

1. 每次修改后需要运行 `npm run build` 验证无类型错误
2. 修改状态相关逻辑时注意不可变更新
3. 动画相关修改注意性能优化，避免每帧创建新对象
4. 保持现有游戏逻辑完整性，避免破坏其他功能
